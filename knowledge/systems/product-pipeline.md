# Product Pipeline

The product release lifecycle every property moves through — from first idea to mature, revenue-generating product.

See [properties/_index.md](../../properties/_index.md) for current phase assignments.

---

## Build

Getting the product from concept to something usable.

| Phase | Description |
|---|---|
| **Idea** | Concept exists but no code yet. Problem and audience are being defined. |
| **Prototype** | First working version. Core mechanic is being explored and validated. |
| **Dogfood** | Usable enough for internal/personal use. Being used regularly to find rough edges. |

## Launch & Validate

Putting it in front of real people and proving it has value.

| Phase | Description |
|---|---|
| **Public Page** | Has a public-facing page or listing. People can discover it and understand what it does. |
| **Signup** | People can sign up or request access. Collecting interest or beta users. |
| **Beta Live** | Real users are actively using the product. Gathering feedback and iterating. |
| **Prod Live** | Stable, public release. Core experience is solid and reliable. |

## Grow

Building awareness and pulling people in.

| Phase | Description |
|---|---|
| **Marketing** | Active marketing efforts underway (SEO, social, content). |
| **Content** | Ongoing content production driving organic traffic. |
| **Lead Gen** | Structured lead generation (email capture, funnels, partnerships). |
| **Sales Funnel** | End-to-end funnel converting visitors to users/customers. |

## Scale

Turning usage into revenue and sustaining growth.

| Phase | Description |
|---|---|
| **Paying** | Users are paying. Revenue exists. |
| **Growing** | Revenue and user base are actively growing. |
| **Mature** | Stable, profitable product with predictable growth. |
