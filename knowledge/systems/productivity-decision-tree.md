# Productivity Decision Tree

## Purpose

When you're stuck, overwhelmed, or starting a new work block, this system tells you which productivity technique to reach for based on your current situation. It solves the meta-problem: "I have 7 frameworks and no idea which one applies right now."

Don't think about this tree. Just follow it from the top.

## The Tree

```
START: What's going on right now?
│
├─ "My brain is full / too many open loops"
│   → GTD Capture. Dump everything into Milou or a notebook.
│     Don't organize yet — just get it out of your head.
│     See: [GTD](../playbooks/gtd.md)
│
├─ "I have too many things in progress"
│   → Personal Kanban. Enforce the WIP limit.
│     Finish or kill something before starting anything new.
│     See: [Personal Kanban](../playbooks/personal-kanban.md)
│
├─ "I don't know what to work on"
│   → Now/Next/Later. Look at your Now column.
│     If Now is empty, pull the top item from Next.
│     If everything feels equal, check your weekly theme.
│     See: [Now/Next/Later](../mental-models/now-next-later.md)
│
├─ "I know what to do but I'm avoiding it"
│   → Rule of 100. Stop thinking, start doing reps.
│     The first rep is the hardest. Just do one.
│     See: [Rule of 100](../playbooks/rule-of-100.md)
│
├─ "I need to plan a feature or project"
│   → Shape Up. Set the appetite first (how much time is
│     this WORTH?), then shape the scope to fit.
│     See: [Shape Up](../playbooks/shape-up.md)
│
├─ "I need to plan the week"
│   → Monday Zoom-Out / Sunday Review.
│     Check bucket balance, assign weekly themes,
│     pick the week's bet, produce the role menu.
│     See: [Time Allocation](time-allocation.md)
│
├─ "I'm burning out on this type of work"
│   → Lion Energy. Rotate roles.
│     Builder fatigue? Switch to Marketer or Strategist.
│     The variety IS the recovery.
│     See: [Lion Energy](../mental-models/lion-energy.md)
│
├─ "How much should I invest in this?"
│   → Investment Tiers. What tier is it? Invest accordingly.
│     Don't spend Tier 3 effort on a Tier 1 experiment.
│     See: [Investment Tiers](investment-tiers.md)
│
├─ "I need to decide whether to keep or kill this"
│   → Tiny Bets + BuJo Migration.
│     Is there signal? Keep it. No signal after fair exposure?
│     Kill it — no guilt. If you keep migrating it forward
│     without doing it, that's your answer.
│     See: [Tiny Bets](../mental-models/tiny-bets.md),
│          [Bullet Journal](../playbooks/bullet-journal.md)
│
└─ "End of day / end of week — time to reflect"
    → Bullet Journal rapid log + GTD weekly review.
      What shipped? What's stuck? What got cut?
      Migration: carry forward what matters, strike what doesn't.
      See: [Bullet Journal](../playbooks/bullet-journal.md),
           [GTD](../playbooks/gtd.md)
```

## Steps

1. **Pause.** Before starting a maker block, take 30 seconds to identify your current state.
2. **Walk the tree.** Find the branch that matches. Don't overthink it — the first match is usually right.
3. **Follow the prescribed action.** Each branch points to a specific technique. Use that technique, not a blend of three.
4. **Return to work.** The tree is a 30-second decision aid, not a 15-minute planning session.

## Triggers

- **Start of each maker block** (5:30 AM, 7:30 PM) — "What do I need right now?"
- **When stuck mid-session** — pause, walk the tree, adjust
- **Sunday weekly review** — run through the tree at a higher level: "What's my dominant state this week?"
- **Milou prompt** — Milou can surface this as a "what do you need?" selector
